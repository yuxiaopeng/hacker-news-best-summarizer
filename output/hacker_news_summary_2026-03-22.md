## 2. 有些事需要时间

**原文标题**: Some things just take time

**原文链接**: [https://lucumr.pocoo.org/2026/3/20/some-things-just-take-time/](https://lucumr.pocoo.org/2026/3/20/some-things-just-take-time/)

阿明·罗纳赫（Armin Ronacher）的文章《有些事就是需要时间》提出，真正的价值和品质往往是内在时间投入的产物，这类似于种树或拥有老旧房产——这些都无法一蹴而就或瞬间复制。这种“嵌入时间”创造了不可替代的价值，就像我们本能地理解瑞士手表的价值或事物需要时间才能成熟一样。

然而，作者观察到现代软件开发和初创企业中存在一种截然相反的趋势：痴迷于即时满足、快速迭代和迅速部署。罗纳赫认为，这种速度可能是有害的，他指出“摩擦”往往具有至关重要的作用。诸如合规性（例如SOC2）或决策冷却期之类的流程，旨在确保质量、周全性和长期可行性。自动化消除所有摩擦，尤其是在人工智能速度的驱动下，会导致跳过诸如评审和健壮基础设施设计等关键步骤，最终将软件的“保质期”从数十年缩短至短短数月，并侵蚀了用户和客户的信任。

此外，人工智能和代理工具所带来的所谓省时益处是虚幻的；任何节省出来的时间都会因竞争压力而迅速被重新填满，导致用于高质量工作的时间反而减少。罗纳赫强调，成功公司和开源项目的决定性因素是坚韧不拔——即能够长期投入、建立关系和培养社区的能力。信任、品质和社区无法一蹴而就；它们需要通过持续的努力和深厚的根基来培养，就像一棵屹立数十年的大树。

## 8. 日本筷子禁忌词汇表 (2022年)

**原文标题**: A Japanese glossary of chopsticks faux pas (2022)

**原文链接**: [https://www.nippon.com/en/japan-data/h01362/](https://www.nippon.com/en/japan-data/h01362/)

这篇2022年的文章《日本筷子禁忌词汇表》概述了*kiraibashi*，即在日本使用筷子时应避免的各种用餐礼仪违规行为。该词汇表按日语假名顺序排列，解释了许多被视为不礼貌或禁忌的行为。

有两项行为尤其严重禁忌：
1.  **合箸 (Awasebashi)**：直接用筷子在两双筷子之间传递食物。这是被禁止的，因为它模仿了葬礼上处理火化遗骸的仪式。
2.  **立箸 (Tatebashi)**：将筷子垂直插入米饭碗中。这是禁忌，因为它类似于佛教的祭祀供品。

其他常见的失礼行为包括：
*   **卫生方面**：洗箸 (Araibashi，在汤中清洗筷子)、咬箸 (Kamibashi，咬筷子)、舔箸 (Neburibashi，舔筷子)。
*   **不敬的拿法**：拜箸 (Ogamibashi，双手合十持筷感谢)、刺箸 (Sashibashi，用筷子戳食物)、敲箸 (Tatakibashi，用筷子敲碗碟)、指箸 (Sashibashi，用筷子指人)。
*   **不佳的餐桌礼仪**：迷箸 (Madoibashi，筷子在菜肴上空徘徊不定)、寄箸 (Yosebashi，用筷子拉动碗碟)、直箸 (Jikabashi，用个人筷子夹取公用食物)、擦箸 (Kosuribashi，过度摩擦一次性筷子)。
*   **不当的放置**：渡箸 (Hashibashi，将筷子像桥一样架在碗碟上)。

这篇文章作为一份全面的指南，旨在帮助人们理解并避免与筷子礼仪相关的文化失误，从而在日本推广一种相互尊重的用餐体验。

## 10. Ubuntu 26.04 终结 46 年的静默 sudo 密码

**原文标题**: Ubuntu 26.04 Ends 46 Years of Silent sudo Passwords

**原文链接**: [https://pbxscience.com/ubuntu-26-04-ends-46-years-of-silent-sudo-passwords/](https://pbxscience.com/ubuntu-26-04-ends-46-years-of-silent-sudo-passwords/)

Ubuntu 26.04 LTS "Resolute Raccoon," due April 23, 2026, will introduce a significant change: `sudo` password prompts will now display asterisks (`*`) for each keystroke, breaking a 46-year tradition of silent input. This "UX fix" has ignited a fierce debate within the Linux community.

Historically, `sudo` prompts showed no visual feedback, a deliberate security measure since 1980 to prevent "shoulder surfing" by concealing password length. This behavior remained untouched across Linux distributions for decades.

The catalyst for this change is `sudo-rs`, a Rust rewrite of the classic `sudo` utility, which Ubuntu adopted as default in version 25.10. Upstream `sudo-rs` recently enabled `pwfeedback` by default, a patch subsequently integrated into Ubuntu 26.04 development builds.

Critics argue this compromises security by exposing password length. However, developers contend that the security benefit of hiding length is negligible, as an observer close enough could still count keystrokes or see graphical login screens (which already show dots). They prioritize a better user experience, particularly for newcomers who might find a blank prompt confusing, confirming input is being registered.

For those who prefer the traditional silent prompt, the behavior can be reverted by adding `Defaults !pwfeedback` to the `sudoers` file using `visudo`. This change takes effect immediately.

The `sudo-rs` implementation is part of a broader modernization effort in Ubuntu 26.04, which includes GNOME 50 on Wayland, Linux kernel 7.0, and increased adoption of Rust-based core utilities, emphasizing memory safety and modern UX sensibilities. The developers have chosen to favor a clearer default experience for new users, while providing an easy option for veterans to maintain the classic behavior.
