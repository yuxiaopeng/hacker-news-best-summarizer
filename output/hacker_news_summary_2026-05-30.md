## 2. 死的经济理论

**原文标题**: The dead economy theory

**原文链接**: [https://www.owenmcgrann.com/p/the-dead-economy-theory](https://www.owenmcgrann.com/p/the-dead-economy-theory)

文章介绍了“死亡经济理论”，该理论认为，在人工智能领域投入的巨额资金，已达数千亿美元并预计将达到数万亿美元，其唯一合理的解释是旨在取代全球劳动力市场。尽管被宣传为“助手”，人工智能潜在的财务模型必然要求在文明层面上消除人力成本中心。

这造成了三阶段的经济崩溃：公司为削减成本、提振股价而解雇员工；然而，被解雇的员工失去收入，导致消费者支出减少，整体需求萎缩；最终，公司发现其客户群已被清除，导致收入停滞。这种“人工智能裁员陷阱”是一个囚徒困境，单个公司为了利润使自动化合理化，但它们却集体摧毁了市场。

与过去较慢且针对特定任务的自动化不同，通用人工智能正全面而迅速地威胁着认知劳动。这给民主国家带来了前所未有的政治危机：公民的影响力，源于劳动、税收和消费，被剥夺。这削弱了税基，使集体谈判变得无关紧要，并加速了财富集中，而所有这些都发生在公共资助的研究惠及私人企业的同时。

人工智能行业领袖承认这一对民主的威胁，但并未提供解决方案。作者指出，一些科技界人士普遍认为民主阻碍进步，导致他们试图规避民主监督。因此，拥有集中决策且无需对选举负责的专制政权成为人工智能更具吸引力的客户，这解释了硅谷转向支持威权政治人物的趋势。文章驳斥了普遍基本收入或“休闲经济”等提议的解决方案，认为它们存在历史性缺陷，并警告经济功能丧失将导致社会崩溃和“绝望之死”。

## 6. SQLite就是你持久化工作流所需的一切。

**原文标题**: SQLite is all you need for durable workflows

**原文链接**: [https://obeli.sk/blog/sqlite-is-all-you-need-for-durable-workflows/](https://obeli.sk/blog/sqlite-is-all-you-need-for-durable-workflows/)

SQLite被视为一种理想的、常被忽视的解决方案，用于构建持久性工作流，挑战了复杂分布式数据库总是必要的这一普遍假设。文章强调了SQLite的核心优势——其ACID（原子性、一致性、隔离性、持久性）特性、单文件架构以及嵌入式、无服务器的本质——这些使其成为确保应用程序健壮性和状态完整性的完美选择，即使面对崩溃也能保持。

对于持久性工作流，SQLite无需独立数据库服务器的运维开销，就能实现应用程序状态存储、任务队列管理以及可靠流程的实施。其事务能力是关键：一个工作流的整个工作单元可以封装在单个事务中，保证所有更改要么全部提交，要么全部不提交，从而防止状态不一致。

详细阐述的具体用例包括管理长时间运行的任务、确保幂等性、处理重试以及通过日志提供可观测性。通过将数据库操作视为工作流状态的单一真理来源，应用程序能够从故障中优雅地恢复。文章将SQLite定位为许多场景下的“恰到好处”解决方案，它比简单的基于文件的存储提供了卓越的可靠性，并且比分布式数据库的复杂性显著降低，使其成为开发弹性软件的强大工具。

## 7. GitHub 封禁发布 Windows 零日漏洞利用的安全研究员

**原文标题**: GitHub bans security researcher who posted zero-day Windows exploits

**原文链接**: [https://www.tomshardware.com/tech-industry/cyber-security/microsofts-github-bans-security-researcher-who-posted-zero-day-windows-exploits-because-company-ruined-their-life-expert-claims-action-is-vindictive-and-promises-further-retaliation](https://www.tomshardware.com/tech-industry/cyber-security/microsofts-github-bans-security-researcher-who-posted-zero-day-windows-exploits-because-company-ruined-their-life-expert-claims-action-is-vindictive-and-promises-further-retaliation)

GitHub has banned security researcher Nightmare-Eclipse (aka Chaotic Eclipse) from its platform, allegedly at Microsoft's behest, amidst an escalating dispute over zero-day Windows exploits. Eclipse claims Microsoft's actions, which also included deleting their Microsoft account, are vindictive. The researcher alleges Microsoft refused communication attempts, did not pay bug bounties for reported vulnerabilities, and caused financial harm, stating they were personally threatened by Microsoft.

Eclipse has an impressive track record, having published six significant Windows zero-day exploits, including BlueHammer, RedSun, and UnDefend—some of which are actively exploited in the wild. These vulnerabilities affect critical components like Defender, CTFMon, and BitLocker. Eclipse vows a "reckoning" on July 14th, threatening to publish more exploits.

Experts suggest a decline in the Microsoft Security Response Center's (MSRC) effectiveness, possibly due to cost-cutting measures replacing skilled staff with "flowchart followers" and introducing rigid requirements like video proof for exploits. Microsoft has remained silent on the matter. Critics argue the ban creates poor optics for Microsoft and does nothing to improve security, as the exploit code is already public. The situation underscores the need for software companies to adapt their security policies in an era of rapid, AI-powered vulnerability discovery.
