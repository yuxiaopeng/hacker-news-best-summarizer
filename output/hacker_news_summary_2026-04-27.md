**原文标题**: The West forgot how to make things, now it’s forgetting how to code

**原文链接**: [https://techtrenches.dev/p/the-west-forgot-how-to-make-things](https://techtrenches.dev/p/the-west-forgot-how-to-make-things)

文章警告说，西方在制造业和国防领域曾通过优化淘汰关键专业知识而重蹈覆辙，现在由于人工智能，这一现象正在软件行业发生。例如，为乌克兰重启“毒刺”导弹生产，因工程师退休和设备陈旧而拖延了数年，这凸显了在数十年的成本削减和整合之后，制造业能力和韧性供应链的丧失。同样，欧洲供应炮弹的努力也因生产线关闭和缺乏批量生产规划而受阻。

“雾堤”（Fogbank）的例子进一步说明了这一点：一种机密核材料无法复现，因为原始专业知识的持有者已退休，记录不完整，并且关键的“未被察觉的依赖关系”已丧失。重建这种复杂能力需要数年时间，而不仅仅是金钱。

作者认为，软件行业正在遵循这一“剧本”。随着人工智能被定位为人类人才的替代品，初级开发人员招聘减少，大学相关专业入学率下降，对人工智能辅助工具的依赖日益增长。这导致了一代人拥有“人工智能辅助能力”，但缺乏修正人工智能或调试复杂系统所需的深入理解、默示知识和判断力。当目前的资深工程师退休时，这种关键知识可能会消失，从而导致“代码雾堤”的局面。西方在国防领域押注于优化导致了危机；同样，在软件工程领域进行同样的押注，也可能带来相似的未来代价。

## 5. GoDaddy 未经任何文件证明，将一个域名转让给了陌生人。

**原文标题**: GoDaddy gave a domain to a stranger without any documentation

**原文链接**: [https://anchor.host/godaddy-gave-a-domain-to-a-stranger-without-any-documentation/](https://anchor.host/godaddy-gave-a-domain-to-a-stranger-without-any-documentation/)

GoDaddy莫名其妙地从客户账户中转移了一个拥有27年历史的域名HELPNETWORKINC.ORG，致使一家全国性非营利组织在20个地点的网站和电子邮件服务中断了4天。尽管该账户设有双重两步验证和域名所有权保护，GoDaddy的一名“内部用户”仍在未经适当验证的情况下完成了转移。

Flagstream Technologies公司的Lee Landis忍受了与GoDaddy支持团队进行的32次电话沟通，累计9.6小时，面对着千篇一律的答复、不断变更的争议处理邮箱地址以及无法关联的案件编号。GoDaddy最终宣布该案件“已结案”，声称已提供了所需文件，迫使Flagstream开始为客户迁移到新域名。

域名得以恢复，仅仅是因为意外接收者Susan意识到自己收到了错误的域名（她原本申请的是另一个），并联系了Flagstream。随后的调查显示，GoDaddy仅凭Susan电子邮件签名中的一个子域名引用就批准了转移，而她并未提交任何支持文件。

此事件暴露了一个严重的安全漏洞，因为恶意行为者本可以利用相同的缺陷。该文章还批评了GoDaddy功能失调的客户支持及其失效的安全披露流程（security@godaddy.com邮箱会退信），敦促GoDaddy公开道歉、进行内部审查并改进沟通。

## 6. 朝日Linux：Linux 7.0 进展

**原文标题**: Asahi Linux Progress Linux 7.0

**原文链接**: [https://asahilinux.org/2026/04/progress-report-7-0/](https://asahilinux.org/2026/04/progress-report-7-0/)

Asahi Linux 团队发布了一份恰逢 Linux 7.0 的进度报告，详细介绍了在各个方面取得的重大进展。

**Asahi 安装器**在两年后获得了首次更新，其特色是实现了通过 GitHub 工作流进行自动化构建和部署。这解决了关键问题，即过时的设备树导致较新的内核（如 6.18+）无法在仅限 UEFI 的安装中启动。此外，新版本还引入了对 Mac Pro 的支持和一种新的固件更新模式。

**环境光传感器 (ALS)** 支持现已可用，能够实现类似原彩显示 (True Tone) 的屏幕调节功能。这需要一个 AOP 驱动程序和一个自动化的安装程序流程，以便从 macOS 中检索并存储校准固件，从而简化未来的更新。

**电源管理**方面取得了显著改进，在 M1 Pro MacBook Pro 等设备上，Pro/Max/Ultra SoC 的空闲功耗降低了约 20%（0.5W）。这通过使电源管理处理器 (PMP) 能够接受电源状态报告来实现。针对基础版 M1 SoC 的工作仍在进行中。

**蓝牙**稳定性得到了改善，消除了扫描期间由 Wi-Fi 干扰引起的音频中断。对博通 (Broadcom) 供应商特定 HCI 扩展的支持现在允许内核优先处理音频流。

最后，在显示控制器 (DCP) 上发现了**可变刷新率 (VRR)** 功能。此前一个被误解的参数被发现可以控制 VRR 适用于外接显示器和 ProMotion 显示器的最小刷新率。虽然目前它需要进行模式设置才能转换状态（这违反了一些标准），但一个内核模块参数 (`appledrm.force_vrr`) 允许用户强制启用该功能。

## 48. macOS 27 网络功能更新

**原文标题**: Networking changes coming in macOS 27

**原文链接**: [https://eclecticlight.co/2026/04/23/networking-changes-coming-in-macos-27/](https://eclecticlight.co/2026/04/23/networking-changes-coming-in-macos-27/)

Apple is giving advance notice of two significant networking changes slated for macOS 27, expected to release in mid-September 2026.

Firstly, **support for AFP (Apple Filing Protocol) is anticipated to be removed.** AFP has been deprecated in favor of SMB for over 12 years. This change will primarily impact users with older Time Capsules or NAS systems that lack SMB3 compatibility. While Macs not upgraded to macOS 27 will retain AFP functionality, Apple silicon Macs updating will lose this support, potentially necessitating new network storage.

Secondly, **future macOS and device OSes will require connections to certain enterprise servers to meet enhanced TLS standards.** These connections, covering services like MDM, DDM, Automated Device Enrolment, app distribution, and Apple software updates, must use at least TLS 1.2 (1.3 recommended), ATS-compliant ciphersuites, and valid certificates meeting ATS standards. Local Content Caching servers are exempt. Checking compliance is complex, requiring a specific network diagnostics logging profile from Apple and analyzing system logs with a provided Terminal command. Apple is gauging enterprise impact, and the timeline for this change could be adjusted based on feedback. Neither change is retrospective.
