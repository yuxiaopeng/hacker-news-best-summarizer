## 1. 美国新拥房措施以人为本

**原文标题**: New US homeownership measure puts people first

**原文链接**: [https://www.minneapolisfed.org/article/2026/new-homeownership-measure-puts-people-first](https://www.minneapolisfed.org/article/2026/new-homeownership-measure-puts-people-first)

本文介绍了一种新的住房自有率衡量标准——房屋所有者人口比 (HPOP)，指出传统的房屋自住率 (65%) 具有误导性，因为它统计的是住房单元，而非个人所有者。HPOP 衡量的是拥有自己住房的成年人所占比例，数据显示全国住房自有率为 53%。这一替代指标为政策考量提供了更细致的视角，通过将所有成年人——甚至包括那些居住在自住房屋中但并非其所有者的人 (占成年人的 13.9%)——纳入考量，更好地反映了经济福祉。

HPOP 为不同人口群体的住房自有率提供了更准确的洞察。例如，HPOP 通过将合住者和大学宿舍居住者考虑在内，显示 35 岁以下年轻人的住房自有率 (22%) 远低于房屋自住率 (37%)。它也为老年人住房自有率呈现了更清晰的趋势。

在州层面，HPOP 揭示了住房成本与住房自有率之间更紧密的关系。住房成本较高的州，例如夏威夷，其房屋自住率和 HPOP 之间的差异显著更大 (18.9 个百分点)。反之，像北达科他州这样住房负担得起的州，差异则要小得多 (3.9 个百分点)，这显著改变了它们的州级排名。作者总结道，HPOP 以人为本的方法提供了对住房状况更深入的理解，将房屋所有权与个人经济现实和当地住房负担能力紧密联系起来。

## 2. 迈向幸福的十步 (2015)

**原文标题**: Ten Steps Towards Happiness (2015)

**原文链接**: [http://hintjens.com/blog:99](http://hintjens.com/blog:99)

彼得·欣琴斯（Pieter Hintjens）的《通往幸福的十个步骤》提供了一份实用的指南，通过有意的行动和观念的转变来培养幸福感，强调幸福不需要金钱或魔法。

这十个步骤倡导：
1.  **投资你的感官**：积极参与并欣赏美食、音乐和大自然等感官体验。
2.  **做你不擅长的事**：勇于学习新技能，感受生命活力，建立自信。
3.  **享受与他人相处**：优先进行社交互动，待人友善，与人交往以建立联系。
4.  **成为更大集体的一部分**：为社区或项目做贡献，感受自身价值，有所作为。
5.  **完成你的项目**：实现目标并记录成就，以建立希望和信心。
6.  **摆脱有害之人**：将那些带来痛苦的负面人物从你的生活中剔除。
7.  **稳固你的情绪**：识别并积极处理愤怒或恐惧等负面情绪，用幸福取而代之。
8.  **重新评估你的时间**：优先考虑有意义的活动和人物，而非浪费时间的承诺，即使这意味着收入减少。
9.  **轻装上阵**：尽量减少物质财产，将其视为负担而非幸福之源。
10. **无欲无求，全盘接受**：培养一种没有欲望或要求的思维模式，拥抱宽容，在每一个瞬间发现美好，以避免失望。

这一框架鼓励一种积极、专注的生活方式，专注于个人成长、有意义的人际关系、情商，并摆脱物质和负面影响。

## 3. 制作

**原文标题**: Making

**原文链接**: [https://beej.us/blog/data/ai-making/](https://beej.us/blog/data/ai-making/)

在布赖恩·“比吉·乔根森”·霍尔的《关于制作》一文中，这位X世代黑客兼计算机科学讲师反思了亲自创造事物所获得的深刻个人满足感，并将其与他发起但由他人或人工智能执行的项目所带来的空虚感进行了对比。

霍尔承认更广泛的“人工智能开发分裂”，但着重探讨“制作的丧失”。他列举了人工智能生成的内容——一段科幻小说节选、艺术作品和代码——以及承包商建造的露台，称他觉得自己并没有“制作”其中任何一样东西。他将这种不适归因于“个人创作”与仅仅“让某物被建造出来”之间的区别，他认为后者缺乏自豪感。

相反，霍尔从他完全亲手编写的一个简单闪存卡系统中获得了巨大的满足感，尽管这比使用人工智能耗时50倍。他认为这才是真正的“制作”。尽管他承认人工智能提示（prompting）的技巧，但他认为这更像是“要求别人制作某物”，而非直接创作。

他探讨了编译器等工具的“灰色地带”：他觉得即使编译器将其翻译成机器代码，或者即使人工智能充当编译器，他仍然是在编写C程序。他认为，关键区别在于直接的“制作”与“请求被制作”，尤其是在使用不精确的人类语言与人工智能交互时，这更像是一种管理者与下属的关系，而非工具与用户之间的关系。这种区别对他而言，对其成就感至关重要。

## 4. 广播变成这样，真是可惜。

**原文标题**: It's a shame what's happened to radio

**原文链接**: [https://blog.jimgrey.net/2026/07/21/its-a-shame-whats-happened-to-radio-3/](https://blog.jimgrey.net/2026/07/21/its-a-shame-whats-happened-to-radio-3/)

一位曾是电台DJ的吉姆·格雷（Jim Grey）在一篇2009年更新的文章中，哀叹广播电台的急剧衰落，称其无论是收听还是工作，都不再令人愉快。他回忆起自己在特雷霍特（Terre Haute）的大学电台WMHD以及专业电台WBOW和WZZQ九年（1985-1994）的职业生涯，在那里他磨练了主持技巧，并珍视通过听众点歌和互动来与他们建立联系的时光。

格雷将广播电台的现状归因于几个因素。首先，电台现在狭隘地针对广告商青睐的人群（例如，“三十多岁的妈妈”），这导致节目形式重复且疏远了更广泛的听众。其次，政府放松管制使广播电台成为由大公司主导的庞大业务，这些公司优先考虑削减成本而非节目质量。这用卫星传输内容和“人声追踪”（voicetracking）取代了现场的本地人才，导致许多电台无人值守且缺乏个性。第三，收视率测量方式的改变，特别是便携式人员测量仪（Portable People Meter）的出现，导致节目时长缩短和人际互动减少，使得电台感觉就像是“匿名的电器”。

因此，格雷已基本放弃了商业广播，转而使用iPhone听音乐，并从NPR获取新闻。他指出，他曾工作过的所有电台现在都已停播。格雷总结说，广播电台为了效率而牺牲本地联系和节目质量，这种自作自受的无关紧要化是一种深刻的耻辱。

## 6. GigaToken：语言模型分词提速约1000倍

**原文标题**: GigaToken: ~1000x faster Language model tokenization

**原文链接**: [https://github.com/marcelroed/gigatoken/](https://github.com/marcelroed/gigatoken/)

Gigatoken 是一款新型语言模型分词器，其设计目标是比现有解决方案（如 HuggingFace Tokenizers 和 Tiktoken）快达 1000 倍，可实现每秒吉字节（GB/s）的分词速度。它提供了即插即用的替换功能，用户只需进行最少的代码修改即可将其与 HuggingFace 或 Tiktoken 模型集成，尽管使用其原生 API 可以实现最高性能。

该工具支持广泛的 CPU 硬件（现代 x86 和 ARM）和常用分词器。在 AMD EPYC、Apple M4 Max 和 AMD Ryzen 处理器上的基准测试表明，Gigatoken 具有卓越的吞吐量，在 GPT-2、Llama、Qwen 等各种基于 BPE 的分词器上，其速度始终比竞争对手提高数百到一千多倍。例如，在 AMD EPYC CPU 上，GPT-2 分词速度达到 24.53 GB/s，比 HF 分词器快 989 倍。

Gigatoken 通过激进的优化实现了其高速性能，包括利用 SIMD 进行预分词、最小化分支、高度优化的预分词映射缓存、减少 Python 交互开销以及避免线程间通信。尽管目前某些分词器（尤其是基于 SentencePiece 的分词器）的优化程度较低，但该项目强调准确性，在兼容模式下其输出与现有分词器匹配。

安装十分简单（`pip install gigatoken`），并且它包含一个用于基准测试和验证的命令行工具。已知问题包括 Python ABI3 开销、缺少文件汇以及对 SentencePiece 和 WordPiece 的优化不完整。其代码库主要是手工编写的，后期在特定优化和 API 改进方面获得了 AI 辅助。

## 8. AI实验室在烧钱扩张吗？

**原文标题**: Are AI Labs Pelicanmaxxing?

**原文链接**: [https://dylancastillo.co/posts/pelicanmaxxing.html](https://dylancastillo.co/posts/pelicanmaxxing.html)

本文探讨了AI实验室是否正在进行“鹈鹕优化”（pelicanmaxxing）——即为西蒙·威利森著名的“骑自行车的鹈鹕”SVG生成基准优化其模型。

作者迪伦·卡斯蒂略进行了一项实验，使用48种动物与交通工具的组合（包括基准提示词），在七个前沿模型（包括GPT-5.6 Terra、Claude Sonnet 5、Gemini 3.5 Flash）中生成了1,008张SVG图像。每张图像都被渲染，由一个LLM裁判（GPT-5.6 Luna）根据动物、交通工具和连贯性进行1-5分的评分，并分析了场景元素。

研究结果很大程度上否定了“鹈鹕优化”现象：
1.  **视觉检查：** 没有发现任何组合（包括鹈鹕-自行车）在视觉上对任何模型表现出持续的优越性。
2.  **动物排名：** 鹈鹕在8种动物中排名第6，落后于猫、鲸鱼和浣熊，表明它们并未受到特殊优先关注。
3.  **交通工具排名：** 自行车排名倒数第二，相对于更简单的交通工具表现不佳。
4.  **难度调整分析：** 一项考虑了固有难度的统计回归分析显示，在所有模型中，鹈鹕、自行车或特定的鹈鹕-自行车组合都没有获得显著的“提升”。Gemini 3.5 Flash模型中关于自行车的微弱信号被认为是假阳性。
5.  **记忆化场景：** 尽管所有21张鹈鹕-自行车图像都朝向右侧，但这种方向性对于单独的鹈鹕和自行车来说都很常见，并且在其他强烈的方向性偏好中并不突出。没有独特的重复场景元素表明存在记忆化。

局限性包括使用单一的LLM裁判和有限的预算。作者总结道，“几乎没有证据表明AI实验室正在进行‘鹈鹕优化’。”相反，对良好SVG生成能力更合理的解释是普遍的“SVG优化”（SVGmaxxing），即实验室优化的是整体SVG生成能力，而非特定的基准。
