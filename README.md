## 2. 在 Oxide 使用 LLM

**原文标题**: Using LLMs at Oxide

**原文链接**: [https://rfd.shared.oxide.computer/rfd/0576](https://rfd.shared.oxide.computer/rfd/0576)

这份Oxide的RFD（讨论请求）概述了公司内部使用大型语言模型（LLM）的指导方针，强调LLM虽然是强大的工具，但其使用必须符合公司的核心价值观。

**核心价值观：**
*   **责任：** 最重要的价值观。员工对所有产出物负全部责任，即使有LLM辅助；人类的判断力仍至关重要。
*   **严谨：** LLM可以提升思维敏锐度，但若使用不慎，也可能用“生成物残渣”取代思考。
*   **同理心：** 对人类读者和作者保持同理心。
*   **团队合作：** 使用LLM不应损害信任。披露LLM在生成作品中的使用，无形中会削弱个人责任感。
*   **紧迫性：** 虽然LLM能带来速度，但速度绝不能损害责任、严谨、同理心或团队合作。

**LLM的用途：**
*   **阅读者：** 非常适用于从大量文档中进行总结和回答问题。注意事项包括数据隐私（确保模型不会用上传数据进行训练），以及在预期需要人工阅读的场景中不能取代人工阅读。
*   **编辑：** 在后期阶段为结构和措辞提供反馈时很有用，有助于保持个人风格。警惕在流程初期过早使用。
*   **作者：** 通常不鼓励使用。LLM生成的散文往往陈词滥调，损害原创性，并削弱作者与读者之间智力投入的社会契约。Oxide珍视员工的写作技能。
*   **代码审查员：** 对特定问题有帮助，但不能取代人工审查。
*   **调试员：** 可以充当“橡皮鸭”来启发问题，有时效果出奇，但不能完全依赖。
*   **程序员：** 对于实验性或“一次性”代码非常有效。对于核心系统需要更谨慎。工程师对LLM生成的代码负全部责任，在同行评审前需要严格的自我审查。不鼓励在同行评审期间进行大规模重新生成。

**结论：** Oxide总体上鼓励使用LLM，但前提是必须严格遵守公司对其产品、客户以及彼此之间深切的责任感。

## 5. Tiny Core Linux: a 23 MB Linux distro with graphical desktop

**原文标题**: Tiny Core Linux: a 23 MB Linux distro with graphical desktop

**原文链接**: [http://www.tinycorelinux.net/](http://www.tinycorelinux.net/)

Tiny Core Linux, from The Core Project, is a highly modular, ultra-small graphical desktop operating system. Its core philosophy centers on a minimal foundation: "Core" (11MB) consists of the Linux kernel and `core.gz`, providing the base for customized desktops, servers, or appliances.

"TinyCore" is a 16MB example built upon Core, adding essential GUI components like Xvesa, Xprogs, and the FLTK/FLWM desktop environment. For an easier start, "CorePlus" offers pre-packaged community extensions, allowing users to choose their preferred desktop while maintaining the modularity principle.

Designed to be nomadic and boot extremely fast, Tiny Core Linux typically resides entirely in RAM, though additional applications can be mounted or installed persistently. It supports booting from CD, pendrive, or hard drive. The system provides a very minimal X desktop, typically with wired internet, giving users complete control to add applications and hardware support from online repositories or by compiling.

The project's goal is to create a customizable, ultra-small graphical desktop OS. It operates with an open development model, encouraging community involvement in packaging extensions and offering support through its forums and a dedicated team.

## 6. 我如何在中国NanoKVM上发现隐藏麦克风

**原文标题**: How I discovered a hidden microphone on a Chinese NanoKVM

**原文链接**: [https://telefoncek.si/2025/02/2025-02-10-hidden-microphone-on-nanokvm/](https://telefoncek.si/2025/02/2025-02-10-hidden-microphone-on-nanokvm/)

本文详细介绍了NanoKVM，一款来自中国公司矽速科技的紧凑且经济实惠的硬件KVM切换器，旨在用于远程计算机控制，且无需在目标机器上安装软件。尽管它承诺开源代码并具备虚拟键盘/鼠标/显示器仿真等功能，作者仍发现了严重的安全漏洞。

这些问题包括初始发货时带有默认SSH密码、用户界面漏洞（例如没有CSRF保护），以及网页登录的硬编码加密密钥，使得密码解密变得容易。该设备依赖中国的DNS服务器，与矽速科技服务器通信以获取更新，并以明文形式存储身份识别密钥。令人震惊的是，它缺乏软件更新完整性验证，并且预装了tcpdump和aircrack等网络分析“黑客工具”。

最令人担忧的发现是设备中内置了一个微型、未提及的2x1毫米麦克风。借助预装的音频录制工具（amixer，arecord），攻击者可以通过SSH轻松录制音频，潜在地实现实时窃听。物理移除是可能的，但由于麦克风尺寸较小，操作起来具有挑战性。

作者将这些缺陷归因于疏忽和仓促开发，而非恶意意图。然而，其开源特性允许安装自定义固件，为提升安全性提供了一条途径。用户也可以物理移除麦克风，或将其改装为扬声器。文章最后提出疑问，还有多少其他设备可能隐藏着未公开的功能，并指出隐私问题并非中国产品的专属，引用了苹果和谷歌的例子。文中提供了测试麦克风的说明。

德国石勒苏益格-荷尔斯泰因州正在启动一项重大转型，其州政府管理将逐步淘汰微软产品，转而采用开源替代品。数字化部长扬·菲利普·阿尔布雷希特宣布，此举旨在实现更大的数字主权，提高数据保护和安全性，并产生可观的成本节约。
