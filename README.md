胡佛水坝的纪念广场上，其水磨石地板设有一个隐藏的天文纪念碑，由奥斯卡·J·W·汉森设计并于1936年建成。这幅星图利用地球25,772年的轴向进动——即那种缓慢的“摆动”，它会使天北极以及我们的北极星（例如，现在的北极星、古埃及时的天枢、12,000年后的织女星）发生移动——来标记大坝的建成日期。

## 8. Shadcn 单选按钮的过于复杂

**原文标题**: The Overcomplexity of the Shadcn Radio Button

**原文链接**: [https://paulmakeswebsites.com/writing/shadcn-radio-button/](https://paulmakeswebsites.com/writing/shadcn-radio-button/)

作者详细描述了在更新一个网络应用样式时，意外遭遇单选按钮复杂性的一次经历。原本以为会是一个简单的 HTML `<input type="radio">`，结果发现项目使用了 Shadcn 的 `<RadioGroup>` 和 `<RadioGroupItem>` 组件。这些组件出乎意料地复杂，包含 45 行代码和多重依赖（甚至一个简单的圆圈都需要图标库），并且它们是基于 Radix Primitives 构建的。

进一步调查显示，Radix 的实现放弃了原生的单选输入框。它转而渲染一个带有 SVG 圆圈的 `<button>` 元素，并依赖 ARIA 属性（`role="radio"`）来传达其语义。作者指出，这种做法违反了“ARIA 使用的第一条规则”，即优先使用原生的 HTML 元素。只有当该组件位于表单内部时，才会存在一个隐藏的原生 `<input type="radio">`。

作者认为这种过度工程化是不必要的，因为使用 `appearance: none` 等 CSS 属性、`::before` 伪元素和 `:checked` 伪类，原生单选按钮的样式设置已经变得非常简单，无需 JavaScript 或外部依赖。尽管承认组件库的吸引力，文章仍批评这种抽象简单浏览器功能的模式。作者总结道，即使对于一个基本的单选按钮，这种过度复杂性也会导致认知负荷增加、bug 增多、JavaScript 包体积变大，并最终导致网站性能下降。
