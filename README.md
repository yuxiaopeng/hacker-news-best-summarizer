**原文链接**: [https://www.tomshardware.com/tech-industry/cyber-security/on-this-day-in-1988-the-morris-worm-slithered-out-and-sparked-a-new-era-in-cybersecurity-10-percent-of-the-internet-was-infected-within-24-hours](https://www.tomshardware.com/tech-industry/cyber-security/on-this-day-in-1988-the-morris-worm-slithered-out-and-sparked-a-new-era-in-cybersecurity-10-percent-of-the-internet-was-infected-within-24-hours)

1988年11月，康奈尔大学研究生罗伯特·塔潘·莫里斯意外地将蠕虫病毒释放到互联网上，在24小时内影响了大约10%的系统。这一事件是网络安全历史上的一个里程碑，发生在互联网的早期阶段，当时互联网主要通过NSFNET供学术和研究机构使用。

莫里斯蠕虫病毒用C语言编写，针对BSD UNIX系统，并利用电子邮件系统和“finger”程序的漏洞。与病毒不同，它是自我复制和自主运行的，导致广泛的系统减速、消息传递延迟和系统崩溃。虽然并非旨在破坏文件，但其影响是巨大的，导致各机构擦除系统并拔掉网络插头以清除蠕虫病毒。

伯克利、哈佛、普林斯顿和NASA等机构都是受害者。联邦调查局介入调查，最终确认了莫里斯，他因违反1986年的《计算机欺诈和滥用法案》而被起诉。他被处以罚款、缓刑和社区服务。

虽然该事件造成了相当大的混乱，但也敲响了警钟，凸显了互联网的脆弱性，并促使人们改进安全措施。它也表明早期互联网社区普遍存在的幼稚信任感正在逐渐消失。目前的估计显示，该蠕虫造成的损失在10万美元到数百万美元之间。文章最后提到了人工智能蠕虫的最新出现。
