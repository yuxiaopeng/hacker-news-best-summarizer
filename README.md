**原文标题**: Bose is open-sourcing its old smart speakers instead of bricking them

**原文链接**: [https://www.theverge.com/news/858501/bose-soundtouch-smart-speakers-open-source](https://www.theverge.com/news/858501/bose-soundtouch-smart-speakers-open-source)

Bose宣布了一项用户友好的决定，将其SoundTouch智能音箱的API文档开源，以避免在官方云支持结束后这些音箱报废。原定的2026年2月18日支持截止日期也已延长至2026年5月6日。

## 2. Tailwind的创建者裁掉了75%的工程团队。

**原文标题**: Creators of Tailwind laid off 75% of their engineering team

**原文链接**: [https://github.com/tailwindlabs/tailwindcss.com/pull/2388](https://github.com/tailwindlabs/tailwindcss.com/pull/2388)

一个旨在为针对LLM（大型语言模型）优化的Tailwind CSS文档添加`/llms.txt`端点的GitHub拉取请求（PR），在被Tailwind Labs联合创始人Adam Wathan拒绝后，引发了一场激烈的辩论。

该PR由`quantizor`于2025年11月发起，旨在为大型语言模型提供一个纯文本、整合版本的文档。然而，Wathan于2026年1月关闭了该PR，并透露Tailwind Labs在前一天因AI对其业务造成“残酷影响”而解雇了其75%的工程团队。

Wathan解释说，他们文档的流量（这是发现他们商业产品的主要渠道）自2023年初以来下降了40%。他表示，进一步为LLM优化文档可能会进一步减少人工流量，从而危及业务的可持续性以及他们维护该框架的能力。他否认了隐性盈利的指控，澄清说他们根据赞助等级划分的`AGENTS.md`提供的是个人LLM最佳实践，而非文档本身。

社区反应不一。许多人表示失望，认为这项决定“不友好开源”，并质疑公司的优先事项。一些人提出了替代的盈利策略，例如在LLM互动中整合付费产品的提示，或者为AI机器人实施按爬取次数付费的解决方案。另一些人则支持Wathan的立场，强调公司财务稳定的必要性，并敦促社区成员在经济上支持Tailwind Labs。`/llms.txt`作为LLM官方标准的有效性也受到了讨论。

## 3. 吃原型食物

**原文标题**: Eat Real Food

**原文链接**: [https://realfood.gov](https://realfood.gov)

美国正面临严峻的健康危机，糖尿病前期、慢性病和医疗保健支出居高不下，这些都与饮食和生活方式选择直接相关。“食真食物”倡议为此推出“新金字塔”，将“真食物”重新定义为完整、营养丰富、天然的食物选择，这些食物经过最少加工，不含添加糖、工业用油、人工香料或防腐剂。

新金字塔优先考虑三大核心食物类别：
1.  **蛋白质、乳制品与健康脂肪：** 强调来自动物和植物的高质量蛋白质（每日每磅体重0.54–0.73克），并搭配来自鸡蛋、海鲜、肉类、全脂乳制品、坚果、种子、橄榄和牛油果等全食物的健康脂肪。
2.  **蔬菜与水果：** 建议每日摄入3份蔬菜和2份水果，注重选择种类多样、色彩丰富、营养密集的完整食物。
3.  **全谷物：** 鼓励每日摄入2–4份富含膳食纤维的全谷物（例如燕麦、大米、真正的酸面包），同时严格限制高度加工的精制碳水化合物。

这一灵活的框架强调蛋白质和蔬菜是肌肉、新陈代谢和肠道健康的基础。它支持健康脂肪，强烈反对摄入任何添加糖，并强调通过水或无糖饮料保持水分。总体目标是重建一种以健康、科学、透明和个人责任为基础的饮食文化，指导美国人做出持续的、真实的食物选择。

## 5. 糖业影响研究人员，并将心血管疾病归咎于脂肪 (2016)

**原文标题**: Sugar industry influenced researchers and blamed fat for CVD (2016)

**原文链接**: [https://www.ucsf.edu/news/2016/09/404081/sugar-papers-reveal-industry-role-shifting-national-heart-disease-focus](https://www.ucsf.edu/news/2016/09/404081/sugar-papers-reveal-industry-role-shifting-national-heart-disease-focus)

2016年，加州大学旧金山分校的研究人员发表在《美国医学会内科杂志》（JAMA Internal Medicine）上的一项分析，揭露了行业文件，显示糖业在1960年代中期蓄意影响了营养科学。他们的目标是将冠心病（CVD）的责任从糖转移到脂肪和胆固醇，同时淡化糖作为风险因素的作用。

行业记录显示，一个糖业贸易组织在1954年预见到，低脂饮食将显著增加蔗糖的消费量。随着将糖与高胆固醇和甘油三酯（两者都是CVD的风险因素）联系起来的证据浮出水面，该行业委托了“226项目”。他们向哈佛科学家支付了相当于5万美元（按2016年美元计算）的费用，以撰写一篇文献综述，该综述于1967年发表在《新英格兰医学杂志》（NEJM）上。

糖业设定了该综述的目标，提供了文章，并审阅了草稿，然而，他们的资助和参与并未披露。《新英格兰医学杂志》的综述严厉批评了将糖与心脏病联系起来的研究，并淡化了脂肪研究中的局限性。它得出结论认为，减少膳食胆固醇并用多不饱和脂肪替代饱和脂肪是唯一必要的干预措施，坚称血液胆固醇是唯一的显著风险因素，从而回避了糖对甘油三酯的影响。

主要作者克里斯汀·卡恩斯表示，这篇综述深刻影响了公众和科学界对心脏病病因的看法。加州大学旧金山分校的研究人员强调，科学综述应由没有利益冲突的个人进行，并且营养科学领域需要强制性的财务披露，尤其是在越来越多的证据表明添加糖与高血压和心血管疾病（CVD）有关联的情况下。

## 6. 船舶地图

**原文标题**: Shipmap.org

**原文链接**: [https://www.shipmap.org/](https://www.shipmap.org/)

Shipmap.org 是由 Kiln 和 UCL 合作开发的可视化项目，展示了2012年全球货船的航行轨迹及相关联的二氧化碳排放量。用户可以探索商船船队的动态，查看五种船型（集装箱船、干散货船、油轮、气体散货船和汽车船）的二氧化碳排放量（以千吨计）和货运能力等统计数据。该交互式地图提供平移、缩放、时间轴以及显示/隐藏港口、航线和船舶的控件，并支持按船型进行筛选和着色。

该地图由 Kiln 创建，将UCL能源研究所研究的数据可视化。UCL EI 处理了 exactEarth 提供的AIS数据和 Clarksons 提供的船舶特性数据，遵循2014年IMO温室气体研究报告，计算出每小时的二氧化碳排放量。Kiln 使用WebGL在定制的测深底图上渲染了这些数据。该项目由欧洲气候基金会资助。

Kiln 提供航线视图的高分辨率打印版本供销售。该地图也可以响应式地嵌入到其他网站上，但需链接回 Kiln。该地图的定制安装服务可根据需求提供。请注意，2012年初的数据不完整，偶尔出现的船舶陆上移动是由于数据点缺失造成的动画效果。

## 7. 停止末日刷屏，开始末日编程：通过手机终端进行构建

**原文标题**: Stop Doom Scrolling, Start Doom Coding: Build via the terminal from your phone

**原文链接**: [https://github.com/rberg27/doom-coding](https://github.com/rberg27/doom-coding)

"Doom Coding" offers a DIY solution for coding on-the-go, enabling users to build via their smartphone's terminal from anywhere with an internet connection. This productive alternative to "doom scrolling" allows aspiring builders to access their home computer remotely, even across continents.

The setup, achievable in about five minutes, requires:
1.  **A Computer:** Running 24/7 with internet, SSH/Remote Login enabled, and Tailscale and Claude Code installed.
2.  **A Smartphone:** With Tailscale and Termius (a mobile terminal tool) installed.
3.  **A Claude Pro subscription.**

**Overview:** The process involves using Tailscale to create a secure, private network, Termius to access your computer's terminal, and Claude Code for AI assistance.

**Getting Started:**
1.  **Computer Setup:** Disable sleep, enable SSH, install and sign into Tailscale and Claude Code.
2.  **Phone Setup:** Install Tailscale (signing in with the same account) and Termius. Configure a new host in Termius using your computer's Tailscale MagicDNS address, port 22, and your computer's login credentials.
3.  **Connect & Code:** Activate the Tailscale VPN on your phone, select your configured host in Termius, and run `claude` to start coding.

**Troubleshooting** addresses common connection issues related to Tailscale VPN status, computer power/lock state, and specific laptop display configurations.

**Best Practices** include tracking progress with `CLAUDE.md`, previewing web projects via an HTTP server and MagicDNS, viewing databases with a PostgreSQL app, and bookmarking essential development sites on your computer for easy mobile access. The guide encourages community contributions and feedback.
