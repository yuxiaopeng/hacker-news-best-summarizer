## 4. Ruby 4.0.0

**原文标题**: Ruby 4.0.0

**原文链接**: [https://www.ruby-lang.org/en/news/2025/12/25/ruby-4-0-0-released/](https://www.ruby-lang.org/en/news/2025/12/25/ruby-4-0-0-released/)

Ruby 4.0.0 引入了两项主要的全新特性：“Ruby Box”和“ZJIT”。Ruby Box 是一种实验性的隔离机制，允许在单个 Ruby 进程中实现独立的定义（例如，猴子补丁、全局变量），非常适合测试或蓝绿部署。ZJIT 是一种新的实验性即时编译器，取代 YJIT，旨在实现更高的性能和更广泛的贡献，尽管尚未达到生产就绪状态。

Ruby 的并行执行特性 Ractor 获得了显著增强，包括用于改进消息通信的新类 `Ractor::Port`、用于共享 Proc 对象的 `Ractor.shareable_proc`，以及减少全局锁争用和提高 CPU 缓存效率的内部优化。Ractor 预计将于明年退出实验阶段。

语言层面变动包括 `*nil` 不再调用 `nil.to_a`，并且逻辑二元运算符（`||`、`&&`、`and`、`or`）现在可以流畅地接续上一行。

核心类更新丰富：增加了 `Array#rfind` 和 `Array#find`；`Binding` 针对带编号参数新增 `implicit_parameters`；`Enumerator.produce` 现在接受 `size` 参数；`ErrorHighlight` 现在会为 `ArgumentError` 同时显示调用者和被调用者；`Kernel#inspect` 允许通过 `instance_variables_to_inspect` 进行自定义；`Pathname` 和 `Set` 被提升为核心类。

兼容性变更包括：由于 `Ractor::Port` 的引入，移除了 `Ractor` 的多个方法（`yield`、`take`、`close_incoming`、`close_outgoing`）。`ObjectSpace._id2ref` 已被废弃，并且 `Process::Status#&` 和 `Process::Status#>>` 已被移除。CGI 库（转义函数除外）和 `SortedSet` 已从默认 gem 中移除。回溯不再显示内部帧。RubyGems 和 Bundler 已更新至 4.0.3 版本。

英伟达已同意以200亿美元现金从人工智能芯片初创公司Groq收购关键资产，包括推理技术和人才。这标志着英伟达有史以来最大规模的收购，远超其2019年以70亿美元收购Mellanox的此前记录。

该交易被构架为一项Groq推理技术的“非独家许可协议”，Groq创始人兼首席执行官Jonathan Ross、总裁Sunny Madra以及其他高级领导人将加入英伟达，共同推进该许可技术。尽管英伟达获得了Groq的资产和知识产权，但Groq本身将继续作为一家独立公司运营，由其首席财务官Simon Edwards领导，其GroqCloud业务不包括在此次交易中。

Groq由谷歌张量处理单元（TPU）——英伟达GPU在AI工作负载方面的直接竞争对手——的创建者于2016年创立。仅仅三个月前，该公司在筹集7.5亿美元资金时估值达到69亿美元。英伟达首席执行官黄仁勋表示，此次收购旨在将Groq的低延迟处理器整合到英伟达的人工智能工厂架构中，从而扩展其用于AI推理和实时工作负载的平台。

这一战略举动与英伟达通过各种许可协议，利用其庞大现金储备，大力投资AI初创公司和人才的更广泛战略保持一致。该公司此前已达成类似的小规模交易，并对其他以AI为中心的公司进行了大量投资。Groq曾设定今年营收达到5亿美元的目标，这得益于市场对其加速芯片的高需求。

## 6. 凤凰：一个用 Zig 从零编写的现代 X 服务器

**原文标题**: Phoenix: A modern X server written from scratch in Zig

**原文链接**: [https://git.dec05eba.com/phoenix/about/](https://git.dec05eba.com/phoenix/about/)

Phoenix是一个用Zig从零开始编写的新X服务器，旨在成为Xorg的现代化、更简单、更安全的替代品。它尚未准备好投入普遍使用，目前能够在一个现有的X服务器中嵌套渲染简单的硬件加速GLX、EGL或Vulkan应用程序。

其主要目标包括简洁性，支持与现代应用程序（过去约20年）相关的X11协议子集，并通过Linux DRM和Mesa GBM支持最新硬件（过去约15-20年），而无需Xorg的服务器驱动接口。安全性至关重要，它具备自动协议解析、Zig的安全特性以及默认的应用程序隔离。应用程序对于敏感操作（例如屏幕录制）将需要GUI提示，或者需要明确的预启动权限。全局热键需要修饰键，除非另有允许。

Phoenix还旨在改进对现代技术的支持，包括正确的多显示器处理（支持不同的刷新率、VRR、HDR）、内置无撕裂合成（具有更低延迟）以及每显示器DPI等新标准。X11协议将为新功能进行扩展，并且Wayland兼容性也是一个目标。其嵌套显示服务器功能有助于调试和开发。

非目标包括替换Xorg服务器（Xorg将保留更广泛的功能/硬件支持），以及X11的“屏幕”概念、`GrabServer`功能或间接GLX。它偏离了核心X11协议，不实现许多旧的字体操作，并默认将字符串视为UTF-8。Phoenix声称，X11中所谓的“根本问题”通常是Xorg服务器特有的，而非协议本身。
