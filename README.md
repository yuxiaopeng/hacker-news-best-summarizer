## 3. AI采纳与索洛生产力悖论

**原文标题**: AI adoption and Solow's productivity paradox

**原文链接**: [https://fortune.com/2026/02/17/ai-productivity-paradox-ceo-study-robert-solow-information-technology-age/](https://fortune.com/2026/02/17/ai-productivity-paradox-ceo-study-robert-solow-information-technology-age/)

本文将当前人工智能的普及与1987年出现的“索洛生产力悖论”进行对比，该悖论指出，尽管人们寄予厚望，但信息技术革命初期未能提振生产力。如今，尽管有374家标准普尔500指数公司积极提及人工智能，但在宏观经济数据中，广泛的生产力提升尚不明显。美国国家经济研究局的一项研究发现，大多数使用人工智能的高管每周仅投入1.5小时，近90%的人报告称，在过去三年中，人工智能对就业或生产力没有影响。然而，这些公司预计人工智能将在未来三年内带来1.4%的生产力增长。

经济学家托尔斯滕·斯洛克（Torsten Slok）观察到，人工智能与索洛时代的计算机类似，“无处不在，却唯独不见于宏观经济数据中”，除了大型科技公司，人工智能在就业、生产力或利润率方面都缺乏迹象。研究结果存在冲突：麻省理工学院2023年的一项研究表明性能提升了40%，而麻省理工学院2024年的一项研究则预测，未来十年生产力将仅适度增长0.5%，被认为是“令人失望的”。员工的不信任以及公司对人才培养的需求（例如IBM将年轻员工招聘数量增加了两倍）被认为是人工智能采用影响缓慢的原因。

然而，文章指出，信息技术繁荣最终在20世纪90年代带来了生产力的大幅提升。经济学家埃里克·布林约尔夫森（Erik Brynjolfsson）和穆罕默德·埃尔-埃里安（Mohamed El-Erian）认为，类似的逆转可能正在发生，他们援引了就业与GDP增长的脱钩以及近期生产力的跃升作为证据。斯洛克设想人工智能将呈现“J型曲线”，即初期增长放缓，随后出现指数级飙升。至关重要的是，与早期信息技术不同，激烈的竞争使得人工智能工具触手可及。最终，人工智能未来的生产力影响取决于公司如何在多样化的经济领域中广泛而有效地实施和整合这项技术。

## 4. 尺寸乱象

**原文标题**: Sizing chaos

**原文链接**: [https://pudding.cool/2026/02/womens-sizing/](https://pudding.cool/2026/02/womens-sizing/)

文章《尺码混乱》强调了女性服装尺码普遍存在的挫败感和不一致性。虽然儿童和青少年服装尺码通常会考虑到成长，但这种体系对成年女性来说却行不通。中位数的成年女性（20岁以上）穿着18码，然而标准“常规”尺码范围通常止于16码，排除了超过一半的成年女性人口。

这种混乱源于完全缺乏统一的尺码标准；品牌独立定义自己的尺码，导致即使在同一公司内部也存在巨大的不一致。通用的字母尺码模糊不清，而专业的“大码”或“曲线型”尺码系列往往缺乏明确的定义和实体店供应。

虚荣尺码（即相同尺寸的服装尺码标签上的数字变得更小）是一种普遍的营销策略。虽然它通过让女性感觉自己能穿更小尺码来鼓励购买，但它也无意中让制造商能够跟上自20世纪90年代中期以来平均女性腰围增长近4英寸的趋势，尽管这只惠及了部分人群。

从根本上说，服装是为大规模生产而优化设计的，而不是为多样化的女性身材。设计从“理想”的8号样本开始，并通过数学方式进行“放码”，这导致大尺码服装的比例失真。这忽视了大多数女性并非“理想”沙漏型身材，而是通常呈矩形，其腰臀比与许多品牌尺码表中严格执行的不同这一事实。这导致大多数人买到不合身的衣服，从而加剧了消费者的不满。

**原文标题**: Anthropic officially bans using subscription auth for third party use

**原文链接**: [https://code.claude.com/docs/en/legal-and-compliance](https://code.claude.com/docs/en/legal-and-compliance)

Anthropic已正式更新其使用政策，明确禁止将免费版、专业版或Max版套餐的订阅制OAuth令牌用于第三方产品、工具或服务，包括Agent SDK。这些OAuth令牌现仅限直接用于Claude Code和Claude.ai。

其他政策细节包括：现有商业协议适用于Claude Code API的使用；业务合作协议（BAA）将扩展至面向具有零数据保留（ZDR）的医疗保健客户的Claude Code；以及一般使用受Anthropic使用政策的约束，其中专业版/Max版限制假定为个人使用。
