## 6. 诺基亚N900魔改

**原文标题**: Nokia N900 Necromancy

**原文链接**: [https://yaky.dev/2025-12-11-nokia-n900-necromancy/](https://yaky.dev/2025-12-11-nokia-n900-necromancy/)

本文详细介绍了作者雄心勃勃地修复朋友的经典诺基亚N900手机的经历。面对设备报废和旧电池不可靠的问题，作者着手设计并构建一个定制的电源解决方案。最初，一个使用大型超级电容器和电阻的原型成功启动了设备。这促使他用十个小型超级电容器制作了一个紧凑的“电池”组，该电池组可以放入原电池仓并由外部供电。

然而，在实验过程中，设备的内部操作系统损坏了（需要从SD卡启动）。随后，一次尝试通过N900脆弱的micro-USB端口为定制电池供电的操作，导致主板上关键的+5V焊盘受到不可逆的损坏，从而使原始USB功能失效。

作者并未因此却步，而是进行了一次“彻底的替换”，定制了一个USB-C端口并将其安装到N900的机身中。由于空间限制，这个新端口仅用于基本的+5V供电（0.5A，不具备数据传输或电力传输功能），因此需要使用普通的USB-A转USB-C线缆。该项目最终完成了一个3D打印外壳，该外壳整合了一个大型电容器、一个降压二极管以及原电池的电池控制模块（BCM），并直接连接到新的USB-C端口。现在，N900能够成功地从SD卡启动，通过其新的USB-C端口供电，并以一台在线收音设备的形式获得了新生，这展示了解决硬件限制的创造性方案。

## 7. 程序员和软件开发者给工具命名已经离谱了。

**原文标题**: Programmers and software developers lost the plot on naming their tools

**原文链接**: [https://larr.net/p/namings.html](https://larr.net/p/namings.html)

作者认为，软件开发者已经偏离了其工具和库的描述性命名规范，转而倾向于受随机名词、神话或虚构角色启发的任意命名。这一趋势，正如理查德·斯托曼呼吁使用反映功能的“易记名称”所强调的，与其他工程领域（如土木工程或化学工程）形成鲜明对比，后者的名称本身就能传达其目的（例如，“金门大桥”、“2,2,4-三甲基戊烷”）。

历史上，早期的软件工具（如grep、sed）和编程语言（如FORTRAN、SQL）也采用了描述性命名。然而，大约在2010年代，出现了一种文化转变，先是倾向于古怪的，然后是越来越没有意义的名称（例如Viper、Cobra、Melody、Casbin），这种转变可能由开源和创业文化所推动。

这种做法给开发者带来了“认知税”，迫使他们浪费精力解读晦涩的名称，而非专注于实际问题或系统架构。这种累积的精力消耗阻碍了新员工的入职和调试，使解释基础设施等任务变得不必要的复杂。

作者驳斥了常见的理由：营销与库无关；清晰度比“无趣”更重要；“趣味性”带来负面外部性；命名冲突可以通过命名空间或前缀解决。

提出的解决方案是一种文化纠正：通过社会压力和教育回归专业标准。开发者应优先考虑清晰性，按功能命名库，使用复合词，并接受冗长。创造性名称应保留给最终用户产品或吉祥物，而基础设施工具则需要描述性名称。指导性问题应该是：“一位土木工程师会这样命名一个桥梁支撑系统吗？”强调清晰度体现了对用户时间与认知资源的尊重。

**原文链接**: [https://riviantrackr.com/news/rivian-unveils-custom-silicon-r2-lidar-roadmap-universal-hands-free-and-its-next-gen-autonomy-platform/](https://riviantrackr.com/news/rivian-unveils-custom-silicon-r2-lidar-roadmap-universal-hands-free-and-its-next-gen-autonomy-platform/)

Rivian自动驾驶与人工智能日公布了该公司雄心勃勃的自动驾驶战略，该战略侧重于定制硬件、人工智能和统一数据平台。

一项重要发布是Rivian自动驾驶处理器(RAP1)，这是他们的首款自研5纳米多芯片模块。RAP1可提供1600稀疏INT8 TOPS算力，为全新第三代自动驾驶计算机(ACM3)提供动力，标志着Rivian转向自主设计芯片。ACM3将于2026年末在R2平台上首次亮相。尽管R2最初发布时没有激光雷达，但Rivian证实未来会集成激光雷达，以补充摄像头和雷达，打造更丰富的感知堆栈。

Rivian的自动驾驶堆栈由一个自我改进的大型驾驶模型驱动，该模型类似于大型语言模型(LLM)进行训练，其发展轨迹旨在实现点对点、免视以及最终的个人L4级自动驾驶。

对于现有车主，一项重大软件更新将为第二代R1T和R1S带来“通用免手持”功能，该功能将覆盖美国和加拿大超过350万英里的道路。

Rivian还宣布了Autonomy+，一个新的订阅级别，将于2026年初推出，费用为一次性2500美元或每月49.99美元。该功能在所有权转让后仍随车辆保留。

为进一步提升拥车体验，Rivian正在围绕“Rivian统一智能”重组其平台，这是一个用于预测性维护和人工智能工具的数据基础。Rivian助手，一款具备短信功能的下一代语音体验，将于2026年初面向第一代和第二代R1车型推出，R2车型将提供离线版本。此外，人工智能将嵌入到服务和诊断工作流程中，以实现更快、更准确的问题解决。
