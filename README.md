## 3. 尺寸乱象

**原文标题**: Sizing chaos

**原文链接**: [https://pudding.cool/2026/02/womens-sizing/](https://pudding.cool/2026/02/womens-sizing/)

文章《尺码混乱》批判了令人沮丧且不一致的女性服装尺码现状，强调了服装系统在贴合普通女性身材方面的普遍性失灵。儿童服装尺码与成长轨迹一致，而女性服装尺码却与实际身体变化显著偏离。20岁以上成年女性的尺码中位数为18码，然而，由于大多数“标准”尺码范围止于16码，导致超过一半的女性被排除在外。

这种混乱源于缺乏统一标准，使得品牌得以制定出极不一致的尺码表。“大码”（Plus）或“曲线型”（Curve）等术语定义不清，且大尺码服装在实体店中往往难以买到。虚荣尺码，即尺码标签上的数字变小而实际服装尺寸却在增大，是一种普遍存在的营销策略，它“蒙蔽”女性对自身真实尺码的认知。尽管这种做法让平均腰围增长时“18码”仍旧有售，但它只让身材偏小的半数成年女性受益。

核心问题在于大规模生产方式。设计师通常以8码（适合不到10%的女性）为起点，然后通过数学计算“放码”出其他尺码。这种统一的比例缩放会扭曲较大体型的比例，且无法适应多样化的身形。业界延续着不切实际的沙漏型身材理想，尽管中位数女性的身材更偏向“矩形”。尺码表中僵硬的腰臀比例导致许多人穿着不合身，迫使作者和许多其他人一样，不得不自己动手制作衣服。

## 4. AI让你无聊

**原文标题**: AI makes you boring

**原文链接**: [https://www.marginalia.nu/log/a_132_ai_bores/](https://www.marginalia.nu/log/a_132_ai_bores/)

文章认为，尽管人工智能是一个有用的工具，但它正在使人们及其项目变得“无聊”，并引用了“Show HN”提交作品质量下降和数量增加的现象。作者认为，人工智能辅助的开发常常导致项目肤浅，因为创作者没有在问题领域投入足够的深入思考或工作，从而缺乏有意义的讨论。这与人工智能时代之前的情况形成鲜明对比，那时人们可以通过交流向那些深入钻研自己工作的人学习。

核心论点是，人工智能让人变得无聊，因为它不擅长原创性思考。将思考外包给大型语言模型会产生缺乏原创性的想法，即使人工智能验证了最初的输入。这对原创工作来说是一个“致命缺陷”。作者驳斥了“人在循环中”的概念，指出原创思想源于那些被外包给人工智能的深入工作，而依赖人工智能实际上会使人类的思维更趋向于人工智能的输出。真正的原创思想源于长期沉浸在某个问题中，然后进行表达——这包括写作或教学等过程，而人工智能的提示无法复制这些。最终，文章总结道，使用人工智能进行思考会阻碍智力发展，就像用挖掘机代替举重会阻碍肌肉增长一样。

**原文链接**: [https://code.claude.com/docs/en/legal-and-compliance](https://code.claude.com/docs/en/legal-and-compliance)

Anthropic 已正式更新其使用政策，明确禁止将来自免费、Pro 或 Max 计划的订阅制 OAuth 令牌用于第三方产品、工具或服务，包括 Agent SDK。这些令牌现在仅限个人在 Claude Code 和 Claude.ai 中使用。

除了认证之外，该文档阐明现有商业协议适用于 Claude API（第一方/第三方）和 Claude Code 的使用。对于已激活零数据保留 (ZDR) 的客户，用于医疗合规的商业合作协议 (BAA) 自动扩展至 Claude Code。所有使用仍受 Anthropic 使用政策的约束，Pro 和 Max 计划的限制基于普通个人使用。安全漏洞可通过 HackerOne 报告。

## 6. 我尝试了完全基于欧洲基础设施搭建我的创业公司。

**原文标题**: I tried building my startup entirely on European infrastructure

**原文链接**: [https://www.coinerella.com/made-in-eu-it-was-harder-than-i-thought/](https://www.coinerella.com/made-in-eu-it-was-harder-than-i-thought/)

作者建立了一家完全基于欧洲基础设施的初创公司，旨在确保数据主权、简化GDPR、避免使用美国超大规模服务商，并支持欧盟科技公司。尽管取得了成功，但这一过程比预期更具挑战性。

所选技术栈包括Hetzner，用于核心计算、虚拟机和S3兼容存储，因为它价格具有竞争力；Scaleway则用于事务性邮件、容器注册表、可观测性服务和域名。斯洛文尼亚公司Bunny.net提供出色的CDN、DNS和安全服务。Nebius负责欧洲GPU计算，用于AI推理；德国提供商Hanko则通过通行密钥和社交登录管理身份验证，确保用户管理层面的欧洲化。

技术栈的很大一部分是自托管的，以实现数据控制和成本效益，其中包括Gitea（源代码控制）、Plausible（分析）、Twenty CRM、Infisical（密钥管理）和Bugsink（错误跟踪），所有这些都运行在基于Rancher的Kubernetes上。Tutanota提供加密电子邮件服务。

主要困难包括寻找具有良好开发者体验且价格有竞争力的事务性邮件服务，从庞大的GitHub生态系统迁移后重建工作流，以及通过欧洲注册商注册某些顶级域名（TLD）时出现难以解释的高价。

关键的是，一些美国依赖是不可避免的：Google Ads和Apple开发者计划对于用户获取和移动应用分发至关重要。Google和Apple等社交登录是转化率的必要条件，尽管Hanko有助于限制其影响。对于前沿AI模型（例如Claude），美国提供商仍然占据主导地位。

尽管由于社区规模较小和文档不完善导致额外付出和进展缓慢，作者仍认为这是值得的。好处包括更低的基础设施成本、明确的数据驻留地以及对技术栈更深入的理解。这一努力表明，构建“欧洲制造”需要在科技行业默认倾向美国服务的情况下，主动选择并付出努力。最终产品是hank.parts。

## 7. HN展示：Micasa – 通过终端追踪房屋

**原文标题**: Show HN: Micasa – track your house from the terminal

**原文链接**: [https://micasa.dev](https://micasa.dev)

Micasa是一款基于终端的工具，旨在帮助用户追踪和管理家庭维护、项目和资产的所有方面。它旨在取代鞋盒和活页夹等杂乱无章的实体系统，提供一个集中化的数字解决方案。

主要功能包括：
*   **维护：** 追踪日程，自动计算到期日期，并记录锅炉滤网等物品的完整服务历史。
*   **项目：** 管理家居装修项目，从构思到完成。
*   **财务与供应商：** 并排比较报价，维护包含联系信息和工作历史的供应商目录。
*   **电器：** 记录购买日期、保修状态和维护历史。
*   **事件：** 记录漏水等问题，标明严重程度和位置，将其与电器或供应商关联，并追踪解决方案。
*   **文档：** 将手册、发票和照片直接附加到项目和电器，所有这些都存储在用户机器上的一个SQLite文件中。

Micasa由键盘驱动，利用Vim风格的模式键进行导航和编辑，其灵感来源于VisiData。通过Go (1.25+) 或适用于Linux、macOS和Windows的预编译二进制文件，安装过程非常简单。用户可以使用`micasa --demo`通过示例数据快速试用。该工具是出于个人需求而开发的，旨在提供一个更高效、更有条理的家庭维护系统。
