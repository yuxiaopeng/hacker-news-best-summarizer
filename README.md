## 1. ytr：Emacs 版 YouTube 广播

**原文标题**: ytr: YouTube Radio for Emacs

**原文链接**: [https://xenodium.com/ytr-youtube-radio-for-emacs](https://xenodium.com/ytr-youtube-radio-for-emacs)

The article introduces "ytr," a new and experimental Emacs package designed to function as a YouTube radio. The author developed "ytr" to fulfill their need for streaming YouTube audio within Emacs, a use case not well-suited for their existing, file-driven "ready-player" package.

Instead of refactoring "ready-player," the author created "ytr" as a separate project to experiment with a widget-like user experience. Users interact with "ytr" by adding a YouTube channel URL; the package then automatically pulls and presents the channel's content metadata in a child frame. It features visual animations, reminiscent of Winamp, which require an Emacs GUI environment.

"ytr" leverages `mpv` and `yt-dlp` for its streaming capabilities. It is currently a brand new, first-iteration package, available on GitHub, and has only been tested on macOS so far. The author encourages users to try it out, provide feedback, and consider sponsoring its continued development.

## 7. Linux 和安全启动证书到期 (2025)

**原文标题**: Linux and Secure Boot certificate expiration (2025)

**原文链接**: [https://lwn.net/Articles/1029767/](https://lwn.net/Articles/1029767/)

使用安全启动（Secure Boot）的Linux用户正面临一个重大挑战，因为微软用于`shim` UEFI引导加载程序的2011年签名密钥将于2025年9月过期。这个密钥对于启用了安全启动的Linux发行版启动至关重要。密钥过期后，新的Linux安装将无法启动，除非其`shim`已使用微软较新的2023年UEFI密钥进行签名。

尽管2023年的密钥已经可用，但许多系统可能尚未安装它。硬件供应商预计将发布固件更新，以将此新密钥添加到系统的数据库中。Linux供应商固件服务（LVFS）及其`fwupd`工具在提供这些更新方面至关重要，无论是作为完整的固件更新还是通过“密钥交换密钥”（KEK）更新。

然而，预计会出现一些复杂情况。一些供应商可能不提供更新，较旧的BIOS系统可能会因EFI变量空间有限而面临更新失败。KEK更新过程也是新的，并且尚未经过大规模测试。对于缺乏更新的系统，禁用安全启动可能成为新安装的唯一选择。

现有的Linux安装应该能够继续启动，因为它们的`shim`已经受信任。然而，对`shim`的任何安全更新都需要新的签名，而微软将不会使用已过期的密钥提供签名，这可能迫使用户运行过时、易受攻击的版本或禁用安全启动。尽管Linux维护者正在努力减轻影响，用户仍应注意可能出现的问题，这凸显了Linux在一个主要以Windows为中心的硬件生态系统中持续面临的困难。

桑格表示，他在长期缺席后于2025年秋季重返维基百科，旨在改革该平台。他目前正在致力于“维基百科知识多元化专题”，这是一个未经批准的倡议，旨在使维基百科对更广泛的贡献者更开放、更受欢迎。

他还重申了其在2002年离开时提出的建议，倡导建立一个开放、包容、中立且专注于良好引用的内容和相互尊重的互动环境的百科全书。桑格自称是一名“包容主义维基人”，并且是“维基百科改革专题”和“反系统性偏见专题”的成员，这与他尽管被封禁但仍推动系统性变革的努力相符。
