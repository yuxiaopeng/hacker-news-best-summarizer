## 1. 安索罗匹克收购邦

**原文标题**: Anthropic acquires Bun

**原文链接**: [https://bun.com/blog/bun-joins-anthropic](https://bun.com/blog/bun-joins-anthropic)

Anthropic 已收购 Bun，这个高性能一体化的 JavaScript 运行时、打包器和包管理器。Anthropic 将把 Bun 集成为其 AI 编程产品（包括 Claude Code 和 Claude Agent SDK）的基础设施，相信它将为未来的 AI 开发工具提供动力。

至关重要的是，Bun 将保持开源、MIT 许可，并继续由原团队在 GitHub 上公开开发。其发展路线图仍将优先考虑高性能 JavaScript 工具、Node.js 兼容性，以及取代 Node.js 作为服务器端运行时。

此次收购带来了多重益处：Bun 将有助于使 AI 编程工具更快、更精简，获得对 AI 编程未来发展的早期洞察，并加速自身发展。对于 Bun 而言，这意味着获得 Anthropic 提供的长期稳定性和资源，使其无需独立将产品商业化。反过来，Anthropic 获得了一个强大的运行时，与 AI 驱动的软件开发不断演进的格局相契合，在这一领域，快速、可预测的运行时对于 AI 智能体编写和部署代码至关重要。

Bun 的发展始于五年前，旨在解决缓慢的开发周期，并发展成为一个迅速广受欢迎的综合性 JavaScript 工具包。其单文件可执行文件已被证明是分发 AI 命令行界面（CLI）工具的理想选择，因此被 Claude Code 等产品采用。此次战略收购确保了 Bun 的持续增长，同时将其定位在 AI 驱动的软件开发前沿。

## 2. Zig 退出 GitHub，称微软的 AI 痴迷毁了该服务

**原文标题**: Zig quits GitHub, says Microsoft's AI obsession has ruined the service

**原文链接**: [https://www.theregister.com/2025/12/02/zig_quits_github_microsoft_ai_obsession/](https://www.theregister.com/2025/12/02/zig_quits_github_microsoft_ai_obsession/)

Zig软件基金会已正式退出GitHub，将其项目迁移至Codeberg，理由是工程卓越性下降和微软对“AI的痴迷”。Zig总裁Andrew Kelly表示，GitHub Actions存在“不可原谅的bug”和“凭感觉的调度”，导致CI系统严重积压。

一个主要例子是2022年2月引入的“safe_sleep.sh”脚本错误，该错误导致CI运行器无限期挂起并占用100%的CPU。这一严重缺陷于2025年4月被公开讨论，但修复方案直到2025年8月才从另一个问题中合并，而最初的bug报告则一直开放到2025年12月。Answer.AI联合创始人Jeremy Howard将这种情况描述为“一系列令人咋舌的低级失误”，表明存在根本性的组织问题。

继Zig离开之后，Dillo浏览器开发者Rodrigo Arias Mallo也宣布计划离开GitHub，原因是担心对JavaScript的过度依赖、潜在的服务中断、可用性下降、审核不力以及GitHub对生成式AI的关注正在“摧毁开放网络”。

此次出走恰逢替代平台Codeberg的用户数量翻倍。尽管GitHub对这些批评保持沉默，微软CEO萨蒂亚·纳德拉仍在强调GitHub Copilot用户的快速增长及其对GitHub收入的巨大贡献，这突显了向AI整合的战略转变，尽管核心开发服务据称正在退化。

## 3. 采用美国汽车标准将危及欧洲人生命

**原文标题**: Accepting US car standards would risk European lives

**原文链接**: [https://etsc.eu/accepting-us-car-standards-would-risk-european-lives-warn-cities-and-civil-society/](https://etsc.eu/accepting-us-car-standards-would-risk-european-lives-warn-cities-and-civil-society/)

巴黎、布鲁塞尔和阿姆斯特丹等多个城市，以及超过75个民间社会组织组成的联盟，正在敦促欧盟立法者放弃其在贸易协议中接受美国较低汽车标准的意图。他们警告说，与美国更宽松的规定对齐将损害欧洲在道路安全、公共健康、气候政策和竞争力方面的全球领导地位。

签署方强调了道路安全方面的鲜明对比：自2010年以来，由于强制性生命安全技术，如行人保护和自动紧急制动，欧盟的道路死亡人数下降了36%。同期，美国的道路死亡人数上升了30%，其中行人和骑自行车者的死亡人数分别增加了80%和50%。欧盟基本的行人保护规则（使特斯拉Cybertruck等车辆非法）将受到威胁，从而毁掉数十年的进步。

公共健康也面临风险。欧盟计划从2026年起对制动器和轮胎磨损产生的有害污染物施加限制，而美国正在放宽车辆的空气污染规则。采纳美国标准将增加欧洲人接触与严重呼吸道和心血管疾病相关的污染物。

从经济角度看，接受美国较低的标准可能会威胁欧洲的就业。主要的欧盟品牌已经在其美国工厂生产符合欧盟标准的车辆。如果欧洲接受美国较低的标准，生产可能会从欧盟转移到美国，导致欧洲汽车工厂及其供应链的大规模失业。这些组织还指出，目前对美国超大型卡车的进口漏洞（已在被滥用）将显著扩大。

签署方敦促欧盟立法者反对这一意图，并公开声明欧盟的汽车标准不容谈判。

## 4. Mistral 3 模型家族发布

**原文标题**: Mistral 3 family of models released

**原文链接**: [https://mistral.ai/news/mistral-3](https://mistral.ai/news/mistral-3)

Mistral AI 推出了其 Mistral 3 系列模型家族，其中包括强大的 Mistral Large 3 和高效的 Ministral 3 系列，所有模型均在 Apache 2.0 许可下发布，旨在促进开放获取并赋能开发者。

Mistral Large 3 是一款最先进的稀疏MoE（混合专家）模型，拥有 410 亿活跃参数和 6750 亿总参数。它为开源模型树立了新基准，在通用提示、图像理解和一流的多语言能力方面表现出色。它首次亮相便在 LMArena 开源非推理模型排行榜上位列第二。

Ministral 3 系列包含 30 亿、80 亿和 140 亿参数的密集模型。这些模型提供了卓越的性价比，具备多模态和多语言能力，并提供基础版、指令微调版和推理版。推理版在其类别中达到了高精度，例如 140 亿参数模型在 AIME '25 上取得了 85% 的成绩。

Mistral AI 与 NVIDIA、vLLM 和 Red Hat 合作，以确保优化的性能和广泛的可访问性。这些模型受益于 NVIDIA Hopper GPU 进行训练，并通过 TensorRT-LLM 和 Blackwell 内核获得高效的推理支持，从而实现从数据中心到边缘设备的部署。

Mistral 3 模型现已在 Mistral AI Studio、Hugging Face、Amazon Bedrock、Azure Foundry 和其他平台上可用。Mistral AI 还提供定制训练服务，以满足量身定制的企业解决方案需求。本次发布强调开放、透明的 AI，具备前沿性能、多模态灵活性、可扩展的效率以及对多样化应用的适应性。

## 7. 国会议员选股表现高出47个百分点。

**原文标题**: Congressional lawmakers 47% pts better at picking stocks

**原文链接**: [https://www.nber.org/papers/w34524](https://www.nber.org/papers/w34524)

NBER最新工作论文《国会山上的“船长收益”》（由魏尚进和周轶帆撰写）揭示，担任领导职务的美国国会议员表现出卓越的选股能力。该研究分析了交易层面的数据，发现这些议员在升任领导职务之前，表现与其同僚相似，但在担任领导职务 *之后*，他们每年平均跑赢同僚47个百分点。

研究人员确定了推动这种卓越表现的两个主要机制。第一个是“政治影响力渠道”，其证据包括：当他们所属党派控制议会时获得更高回报；在监管行动之前策略性地出售股票；以及购买获得更多政府合同或在立法上获得党派有利支持的公司股份。第二个是“企业接触渠道”，体现在股票交易能够准确预测随后的公司新闻并产生更高回报，尤其是来自捐助者拥有的公司或位于其本州的公司。这项研究于2025年11月发表，表明政治权力和特权信息有助于国会领导人的财务成功。

## 10. 苹果发布开源权重视频模型

**原文标题**: Apple Releases Open Weights Video Model

**原文链接**: [https://starflow-v.github.io](https://starflow-v.github.io)

苹果公司发布了STARFlow-V，这是一款基于归一化流（NFs）的开创性视频生成模型，旨在视觉质量上与扩散模型相媲美。这标志着归一化流首次在因果视频生成中展现出这种能力，并提供了端到端训练、精确似然估计以及对文本到视频（T2V）、图像到视频（I2V）和视频到视频（V2V）任务的原生多任务支持等优点。

STARFlow-V采用了一种全局-局部架构，其中一个深度因果Transformer在压缩的潜在空间中处理全局时间推理，而浅层流块则管理局部帧内细节。这种设计最大限度地减少了误差累积。主要创新包括流-分数匹配（Flow-Score Matching），该技术整合了一个轻量级因果去噪器以提高视频一致性；以及视频感知雅可比迭代（Video-Aware Jacobi Iteration），通过并行更新来提高采样效率。

该70亿参数模型使用7000万文本-视频对和4亿文本-图像对进行训练，能够以16fps生成480p视频。其固有的可逆结构使其无需改变架构即可无缝支持各种生成任务。经验证明，STARFlow-V实现了强大的视觉保真度和时间一致性，这使得归一化流成为高质量自回归视频生成和“世界模型”开发的一个极具前景的方向。
